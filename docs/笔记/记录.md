# controller和service层方法命名区分

| 层级       | 推荐名称          | 说明                       |
| ---------- | ----------------- | -------------------------- |
| controller | CreateUserHandler | 表示这是一个API Hander     |
| service    | CreateUserService | 表示这是业务逻辑核心处理   |
| model      | AddUser           | 数据访问层一般保持简洁命名 |
